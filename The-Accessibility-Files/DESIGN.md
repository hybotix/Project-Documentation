# The Accessibility Files — Design

## Principles

- Accessibility is not an afterthought — it is a first-class design requirement
- All Hybrid RobotiX tools and interfaces should be evaluated for accessibility
- Dark Reader compatibility is a baseline requirement for any web-based UI

## Scope

The Accessibility Files covers:
1. Accessibility contributions to third-party projects (e.g., Delta Robotics Protoboard)
2. Internal accessibility standards for Hybrid RobotiX tooling
3. Research and documentation on assistive technology
4. My Chairiet Platform accessibility features (display, UI, interaction)

## Dark Reader Compatibility

Dark Reader compatibility means ensuring all UI elements (contrast, borders, backgrounds) render correctly when Dark Reader's CSS overrides are applied. This was a specific contribution to Delta Robotics' Protoboard PCB design software.
